# Change Request Log

Use this file to control scope changes after the user confirms implementation.

## Rule

After readiness gates are complete and implementation is confirmed, new or changed requirements must be recorded here before implementation changes.

## Impact Assessment

For every change request, assess:

- product scope impact
- implementation effort impact
- test scope impact
- delivery risk
- documentation updates needed

## Decision Values

```text
pending -> approved -> rejected -> implemented -> tested -> accepted
```
