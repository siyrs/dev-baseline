## Highlights

- Compact team task workspace: `00-index.md` plus `01-07` task records.
- PM-led dynamic agent roster with default specialists and PM-defined custom specialists.
- One-line requirements can start intake, but code implementation requires Architect + Developer elaboration before work starts.

- Compact-first scripts for task creation, readiness, traceability, reports, dashboard, metrics, and status events.
- README and README_CN now include the v0.3.0 flow diagrams and compact task document model.

## Team flow changes

- PM can activate the smallest useful set of default specialists: Analyst, Architect, Developer, QA Tester, Coordinator.
- PM can define ad-hoc specialists such as Security Reviewer, Data Migration Reviewer, Performance Reviewer, Release Operator, Documentation Owner, UX Reviewer, or Domain Expert.
- Custom specialists must have an initialization prompt recorded in `03-work-log.md` before activation.
- Custom specialists report only to PM and must not silently change the task contract.

## Requirement elaboration

A one-line request is enough for intake, but not enough for implementation. When code changes are needed, PM should coordinate Architect and Developer to produce a workable plan in `02-delivery-plan.md` covering architecture impact, implementation approach, likely areas/files/modules, constraints, dependencies, risks, self-test, and validation expectations.
